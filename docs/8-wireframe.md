# 와이어프레임 - my_todolist (Team CalTalk)

작성일: 2026-02-11 | 버전: 1.1 | 상태: Draft

---

## 목차

1. 로그인 페이지 (`/login`)
2. 회원가입 페이지 (`/signup`)
3. 할일 목록 페이지 — 기한 초과 항목 있는 상태 (`/`)
4. 할일 목록 페이지 — 정상 상태 (기한 초과 없음) (`/`)
5. 할일 생성 모달 — 빈 폼
6. 할일 수정 모달 — pre-fill 상태
7. 삭제 확인 다이얼로그
8. 로그인 에러 상태 (`/login`) — 잘못된 자격증명
9. 회원가입 에러 상태 (`/signup`) — 중복 이메일 + 유효성 오류
10. 컴포넌트 명세

### 인터랙션 명세

| 요소 | 동작 |
|------|------|
| 검색바 | 입력 시 실시간 필터링 (title, description 부분 일치), X 버튼으로 즉시 초기화 |
| 정렬 드롭다운 | 마감일 오름차순(기본) / 마감일 내림차순 / 등록일 오름차순 / 등록일 내림차순 선택 |
| 상태 필터 탭 | 전체 / 미완료(pending) / 완료(completed) 클릭 시 즉시 목록 갱신 |
| 기한 초과 그룹 | 빨간 테두리 + ⚠ 아이콘 + 그룹 상단 고정 표시, 항목 수 배지 표시 |
| 체크박스 `[ ]` | 1회 클릭 → 완료 처리 (pending → completed), 기한 초과 그룹에서 즉시 제거 |
| 체크박스 `[✓]` | 1회 클릭 → 미완료로 되돌리기 (completed → pending) |
| [✏] 수정 버튼 | 클릭 시 할일 수정 모달 오픈, 기존 값 pre-fill |
| [🗑] 삭제 버튼 | 클릭 시 삭제 확인 다이얼로그 표시 |
| [+ 할일 추가] | 클릭 시 할일 생성 모달 오픈 |
| [로그아웃] | 클라이언트 토큰 삭제 후 `/login`으로 이동 |
| 페이지 로드 중 | `[로딩중...]` 인디케이터 표시 |

### 재사용 컴포넌트 목록

---

#### TodoCard

할일 목록의 개별 항목을 렌더링하는 카드 컴포넌트. 일반 / 기한 초과 / 완료 세 가지 상태를 가진다.

| Props | 타입 | 설명 |
|-------|------|------|
| `id` | `string` | 할일 고유 ID (UUID) |
| `title` | `string` | 할일 제목 |
| `description` | `string \| null` | 할일 설명 (선택) |
| `due_date` | `string` | 마감일 (YYYY-MM-DD) |
| `status` | `"pending" \| "completed"` | 완료 상태 |
| `is_overdue` | `boolean` | 기한 초과 여부 (서버 판정) |
| `onToggle` | `(id: string) => void` | 완료/미완료 토글 핸들러 |
| `onEdit` | `(id: string) => void` | 수정 버튼 클릭 핸들러 |
| `onDelete` | `(id: string) => void` | 삭제 버튼 클릭 핸들러 |

| 상태(State) | 타입 | 설명 |
|------------|------|------|
| `isToggling` | `boolean` | 완료 토글 API 호출 중 여부 |

| 시각적 상태 | 조건 | 스타일 |
|-----------|------|--------|
| 일반 | `status = pending`, `is_overdue = false` | 기본 테두리 |
| 기한 초과 | `status = pending`, `is_overdue = true` | 빨간 테두리 + ⚠ 아이콘 |
| 완료 | `status = completed` | 제목 취소선 + 흐린 색상 |
