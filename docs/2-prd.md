## 1. 제품 개요

### 비전

학생이 복잡한 학사 일정과 과제를 놓치지 않도록 돕는, 단순하고 빠른 개인 할일 관리 도구를 만든다.

### 목적

기존 메모앱이나 노트 앱은 마감일 관리와 완료 추적에 최적화되어 있지 않다. my_todolist는 마감일 중심의 할일 목록과 기한 초과 시각화로 학생의 일정 누락을 방지한다.

### 제품 목표

| 목표 | 세부 내용 |
|------|---------|
| 학생 일정 누락 방지 | 마감 기한 경과 항목을 즉시 식별할 수 있도록 시각 구분 제공 |
| 빠른 할일 등록/완료 | 3단계 이내 할일 생성, 1회 클릭 완료 처리 |
| 안전한 개인 데이터 | JWT 인증으로 타인의 할일 접근 차단 |
| 신속한 MVP 출시 | 2일 이내 핵심 기능 구현 및 배포 |

## 4. MVP 기능 범위

### In Scope (구현 대상)

| 기능 그룹 | 세부 기능 | 우선순위 |
|---------|---------|--------|
| 인증 | 회원가입 (이메일/비밀번호/이름) | P0 |
| 인증 | 로그인 / 로그아웃 (JWT) | P0 |
| 할일 관리 | 할일 생성 (제목, 설명, 마감일) | P0 |
| 할일 관리 | 할일 목록 조회 (마감일 정렬 기본) | P0 |
| 할일 관리 | 할일 수정 (제목, 설명, 마감일) | P0 |
| 할일 관리 | 할일 삭제 | P0 |
| 할일 관리 | 완료/미완료 상태 토글 | P0 |
| 시각화 | 마감 기한 경과 항목 최상단 별도 그룹 표시 | P0 |
| 조회/검색 | 상태 필터 (pending / completed) | P1 |
| 조회/검색 | 정렬 옵션 (due_date, created_at 오름/내림차순) | P1 |
| 조회/검색 | 키워드 검색 (title, description 부분 일치) | P1 |

### Out of Scope (MVP 제외)

| 제외 기능 | 제외 이유 |
|---------|---------|
| 팀/공유 할일 | 멀티 테넌트 구조 필요, 복잡도 높음 |
| 반복 일정 설정 | 스케줄러 구현 필요 |
| 알림/리마인더 (이메일, 푸시) | 외부 서비스 연동 필요 |
| 파일 첨부 | 스토리지 인프라 필요 |
| 캘린더 뷰 (월간/주간) | 추가 UI 구현 비용 높음 |
| 소셜 로그인 (Google, Kakao) | OAuth 연동 복잡도 |
| 모바일 앱 (iOS/Android) | 웹 앱으로 우선 검증 |
| 서버 측 토큰 블랙리스트 | Redis 등 추가 인프라 필요 |

## 6. UI/UX 요구사항

### 주요 화면 목록

| 화면 | 경로 | 접근 조건 | 핵심 요소 |
|------|------|---------|---------|
| 로그인 | `/login` | 비인증 | 이메일/비밀번호 입력, 회원가입 링크 |
| 회원가입 | `/signup` | 비인증 | 이름/이메일/비밀번호 입력, 유효성 안내 |
| 할일 목록 | `/` (홈) | 인증 | 기한 초과 그룹, 할일 카드, 검색바, 필터, 정렬 |
| 할일 생성 | 모달 or `/todos/new` | 인증 | 제목/설명/마감일 입력 폼 |
| 할일 수정 | 모달 or `/todos/:id/edit` | 인증 (소유자) | 기존 값 pre-fill, 수정 후 저장 |

### 화면 흐름

```
[비인증 진입]
    │
    ├─ /login ──────── 로그인 성공 ──────── /  (할일 목록)
    │                                          │
    └─ /signup ─────── 가입 성공 ─ 자동 로그인 ┘

[할일 목록 화면 - /]
    │
    ├─ 기한 초과 그룹 (빨간 테두리/아이콘)
    │     └─ 완료 처리 / 수정 / 삭제
    │
    ├─ 일반 할일 그룹 (due_date 오름차순)
    │     └─ 완료 처리 / 수정 / 삭제
    │
    ├─ 상태 필터 탭 (전체 / 미완료 / 완료)
    ├─ 정렬 드롭다운
    ├─ 키워드 검색바
    └─ 할일 추가 버튼 ─── 생성 모달/폼
```

### UI 요구사항

| 항목 | 요구사항 |
|------|---------|
| 마감 기한 경과 표시 | 빨간 계열 색상 + 경고 아이콘으로 구분, 목록 최상단 그룹 배치 |
| 완료 처리 | 체크박스 또는 토글 버튼 1회 클릭으로 상태 변경 |
| 삭제 확인 | 실수 방지를 위한 삭제 확인 다이얼로그 제공 |
| 반응형 | 모바일 브라우저에서도 기본 사용 가능한 레이아웃 |
| 입력 유효성 | 폼 제출 전 클라이언트 측 유효성 검증 및 에러 메시지 인라인 표시 |
| 로딩 상태 | API 호출 중 로딩 인디케이터 표시 |

## 8. 기술 스택

> 3-티어 아키텍처 (Frontend / Backend / Database) 기반으로 구성

| 계층 | 기술 | 버전 | 비고 |
|------|------|------|------|
| **Frontend** | React | 19 | UI 컴포넌트 기반 SPA |
| | TypeScript | 최신 | 타입 안전성 확보 |
| **Backend** | Node.js | LTS | 서버 런타임 |
| | Express | 최신 | REST API 서버 프레임워크 |
| | pg | 최신 | PostgreSQL 직접 연동 라이브러리 (ORM 미사용) |
| **Database** | PostgreSQL | 17 | 메인 데이터베이스 |

### 아키텍처 구조

```
┌─────────────────┐     HTTP/REST     ┌─────────────────┐     SQL     ┌─────────────────┐
│                 │ ◄────────────────► │                 │ ◄──────────► │                 │
│  React 19 + TS  │                   │ Node.js/Express  │             │  PostgreSQL 17  │
│   (Frontend)    │                   │   (Backend)      │             │   (Database)    │
└─────────────────┘                   └─────────────────┘             └─────────────────┘
```

### 기술 선택 근거

| 항목 | 결정 | 이유 |
|------|------|------|
| ORM 미사용 | `pg` 직접 사용 | SQL 제어권 확보, 불필요한 추상화 제거 |
| React 19 | 최신 버전 사용 | 최신 훅 및 성능 개선 활용 |
| PostgreSQL 17 | 도메인 정의서 DDL과 동일 | UUID, CHECK 제약, 인덱스 전략 그대로 적용 가능 |
